Design

IxD Process

Goal Alignment - The goal for this website was to create a portfolio website which firstly showcases my skills as a web developer, and secondly, showcases who I am. I want users to be able to see who I am and the capabilities I have through the website. The ideal target audience for this website would be companies and recruiters.

Information Structure

a)Home - The home page should give a brief overview of who I am and what the website entails. It should have links to all the other pages.

b)Profile - The profile page should give more information about me. Things such as my interests, skills, education and experience should be shown here.

c)Blogs - The blog page will have all the weeks blogs. Ideally it should be systemized and orderly.

d)Essay - The essay page will have the essay that I will have written.

e)Design - The design page will be the longest and hardest to do. It will need to represent the design process of the website. It should also be orderly and systemized.

f)Portfolio - The portfolio page will showcase a few pieces of work that I have done. It should include images of the work and a short description of what it is.

User Flow

Interface Elements - In terms of interface elements, the main things I will be using are links for the navigation as well as dropdown buttons for the blog posts. At this stage of my website I don’t think I’ll need other elements. .

Style Process

Visual Aesthetic - The visual aesthetic I was aiming for was a minimalistic one. I didn’t want too many colours or things that pop at you, rather, it should be something that’s cool and calming.

Visual Design Elements:

a)Fonts - The fonts I used are simple, yet common fonts. I used Verdana, Geneva, Tahoma and Sans Serif.

b)Readability - Due to using simple fonts, all my text is readable. I have made the font sizes readable as well.

c)Colours - The initial colour scheme for my website was orange and blue, but after some consideration I realized that this wasn’t ideal for what I wanted my website to be. I therefore changed it to blues and greys. This fits in with the minimalistic and cooling aesthetic that I was going for. The text colours are either black or white, which blends in with the colour pattern but that also stands out against it for readability.

d)Colour Theory - The colour palettes I have used are a range of blues and greys. 7DB0CF, 9BAFC5, 8AADC5, B6CCDE, CED6E0. All these colours blend in well together.

Style Guide

Goal & Aesthetic - My goal is to have a website that represents me and that showcases my passions, talents, and character, and I think that the aesthetic of the vibrant, simple, and calming nature of the website represents me well.

Visual Design Elements - The visual design elements I made use of is a simple background with the colour palette mentioned above. In order to highlight the text more I added in a background for the text which is a gradient of two types of blues. My text is either black or white and with a simple font. There is nothing too fancy or extraordinary, but rather, simple and sufficient.

Implementation Notes

Folder Structure:- My folder structure reflects the hierarchical information structure of my content. I used HTML, CSS and JavaScript. All my data and the skeleton structure are in HTML, and the presentation layer or 'clothing' is in CSS. I have a few functions in JavaScript. The naming conventions I used are in a simple to understand yet orderly manner, which will allow me to go back at any time and make any necessary changes. Despite having just one CSS script for all 6 pages of HTML content, using classes and IDs, I was able to keep all my code in order.

Semantic Markup:- Before beginning with this assignment, the semantic markup portion scared me as I wasn't sure if I'd get the hang of it so early on. I was hoping not to have to be one of those that have a million divs in my code, but thankfully, once I started coding, I quickly got the hang of it and was able to semantically code. In my first assignment I used a few divs, but this time around I was able to not use any divs and instead made efficient use of headers, footers, mains, nav, and other relevant tags. I also used a lot of article and section tags which I didn’t make use of previously. All my interface elements are displayed as I wished for it to be without any errors or issues. I understand that for a website to be run efficiently and for it to be search engine optimized, semantics are very important. I have also added in more meta tags which will give the browser more information about my website.

JavaScript:- Having used HTML and CSS since the beginning and using it for the first website, I was well versed with it and understood how to write out what I needed to. However, it was my first time using JavaScript so it was abit of a challenge for me. I used it to write out functions for my navbar as well as for my blog posts section. I made sure to not exceed 8 lines of code for a single function and I was able to link it to my HTML and CSS.

CSS:- I did not use any in-text styling in HTML. All my styling is only done in CSS. With the use of classes and IDs and proper naming conventions, my code is scalable and modular, and if at any time any changes need to be made, it will be very easy to make those changes. I changed all the old measurement units I had as these weren’t relative and didn’t assist with responsive design. I used relative units such as em, vh and vw so that no matter what device the website is viewed on, it will be responsive.

Website Improvement

Assignment 3:

a) Style - The first thing I wanted to change was the aesthetic and colours of the website. Whilst I did like the previous orange and blue colours, I found it to be slightly distracting from the more important text. I also wanted to try and maintain the clean and minimalistic look. For that reason I chose this blueish grey background which is showcased on every page and which allows me to have the important text stand out.

b) HTML - After receiving feedback from the previous edition of the website, I made a few changes to my code. With HTML I got rid of all divs and I added in more semantic markup.

c) CSS - In my previous website, I didn't make use of relative measurement and responsive design. Using CSS, I now changed all my measurements to relative measurements and my website is mostly responsive, with the exception of a few things like the navbar and index page.

d) JavaScript - This was my first time using JavaScript so I wasn't too well versed with it. I added in two functions with JavaScript. The first function is for the blogs. It allowed me to create an accordion style blog page in which you can simply click on a blog to expand or close it. The second function I added in was for the navbar. When you begin scrolling on the website the navbar will remain at the top and it will slightly change its background colour so that it is more visible when shown over other content.
